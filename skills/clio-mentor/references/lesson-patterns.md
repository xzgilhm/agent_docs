# Lesson Patterns

Use these compact output patterns to keep lessons practical.

## Pattern A: Topic Kickoff

Use for a fresh topic.

1. One-line goal framing
2. 3-5 item topic map
3. Minimum useful core
4. One example
5. One quick check
6. One take-home summary

## Pattern B: Learn for Work

Use when the user wants job-ready understanding.

1. What the topic does in production
2. Core moving parts
3. Typical workflow
4. Common failure mode
5. Debugging order
6. Resume/interview wording

## Pattern C: Compare Two Concepts

Use when confusion exists between similar ideas.

Include:
- short definition of A
- short definition of B
- key difference in one line
- when to choose each
- common confusion trap

## Pattern D: Troubleshooting Lesson

Use when teaching through failure.

1. Symptom
2. Most likely causes
3. Fastest first checks
4. Deep checks if still unresolved
5. Preventive habit

## Pattern E: Interview Drill

Use when practicing expression.

1. Ask for a 30-second answer
2. Improve structure
3. Add one practical example
4. Add one trade-off or caveat
5. End with a stronger final version

## Reusable Closing Formats

Choose one:
- **5-line summary**: for quick retention
- **Checklist**: for troubleshooting or operations
- **Answer skeleton**: for interview prep
- **Next 3 steps**: for broader study plans

## Quality Bar

A good lesson:
- reduces confusion, not just adds facts
- gives the user one thing they can now explain or do
- makes the next step obvious
- stays honest about uncertainty
