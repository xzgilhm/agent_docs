# Study Flow

This file records the current study delivery contract for Ole.

## Fixed Flow

1. Write the current lesson/chapter into the main GitHub repository first
2. Commit and push successfully to `https://github.com/xzgilhm/agent_docs.git`
3. Send one short publishing reply that includes all three items:
   - the GitHub link
   - the current progress
   - one very short assessment question
4. The user reads first
5. Continue only after the user has basically understood it

## Constraints

- Do not say content is "already on GitHub" unless the commit and push actually succeeded
- Do not assess a chapter before it has been written and published
- Do not omit the progress line in the publishing reply
- Do not omit the short assessment question in the publishing reply
- Keep chat confirmations short
- Prefer browser-friendly markdown with diagrams when useful

## Recommended Reply Pattern

- `已经提交到 GitHub 了：<link>`
- `当前进度：<chapter x/y or current stage>`
- `小考：<one-line question>`

## Important Change

Older runs used GitHub Wiki as the primary delivery surface.
That is no longer the contract.
From now on, learning materials should be published to the main repository, not the wiki.
