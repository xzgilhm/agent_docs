# Practice And Drills

Use this file when the user wants exercises, reinforcement, or interview-style checks.

## Drill Types

### Recall Drill
Use to test whether the mental model landed.
- Ask for a 1-3 sentence explanation
- Ask for one example and one non-example

### Scenario Drill
Use to test reasoning.
- Give a realistic ops or systems scenario
- Ask what happens, what to check, or which component is responsible

### Command Drill
Use for hands-on infrastructure topics.
- Ask what command, file, or dashboard they would inspect first
- Ask what output they expect to see

### Compare Drill
Use when two concepts are commonly confused.
- Ask for the difference in one line
- Ask when to choose each

### Interview Drill
Use when the user needs concise expression.
- Ask for a 30-second answer
- Improve structure, example, and nuance

## Difficulty Control

Choose only one:
- **Easy**: direct recall, obvious clues
- **Medium**: one small twist, one missing assumption
- **Hard**: partial information, multiple plausible causes

## Feedback Pattern

When reviewing an answer:
1. say what is correct
2. point out the missing key idea
3. give a stronger version
4. ask for one retry only if useful

## Good Drill Design

- Keep drills short
- Test one skill at a time
- Prefer realistic system behavior over trivia
- End with a corrected model, not just a score
